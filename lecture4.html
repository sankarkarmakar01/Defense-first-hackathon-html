What is File Path in HTML? üìÇ

In HTML, a file path tells the browser where to find another file (like an image, CSS, or another page). A website is made of multiple files (HTML, CSS, JavaScript, images, fonts, etc.) and they must connect correctly using file paths.

Types of File Paths (relative, absolute)

  • Relative Path: Directions to a file starting from the current file's location.

  • Absolute Path: The full URL to a resource on the web.

The Purpose: To correctly link files inside a project or to external resources.

Relative File Paths

Relative paths work within your project folder structure.

Parent Folder (Going Up):

Key Point: Use ../ to move up one folder level.

Absolute File Paths

Absolute paths give the complete address of a file on the internet. 

<a href="https://www.google.com">Search on Google</a>

Warning: Never use absolute paths for your local files (like C:/Users/YourName/website/image.png), because they will break when uploaded to a server!

What is HTML Boilerplate? üìù

In HTML, a boilerplate is the standard starting code that every web page needs. It gives the browser essential instructions to properly render the page.

Boilerplate Elements (<!DOCTYPE>, <html>, <head>, <body>)

  • <!DOCTYPE html>: Tells the browser to use HTML5.

  • <html>: Root element of the page.

  • <head>: Contains metadata, title, links, etc.

  • <body>: Contains all visible content of the page.

The Standard HTML5 Boilerplate

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
  </head>
  <body>
    <!-- Your content goes here -->
  </body>
</html>

Breaking Down the Boilerplate

  • <!DOCTYPE html>: Defines HTML5 standard.
  • <html lang="en">: Root element with language set to English.
  • <meta charset="UTF-8">: Ensures proper text encoding.
  • <meta name="viewport">: Makes the site mobile-friendly.
  • <title>: Sets the browser tab title.
  • <body>: Holds visible content.

Pro Tip: Always include the viewport meta tag for responsive design!

What is Page Structure in HTML? üìÉ

In HTML, semantic structure organizes content meaningfully using tags like <header>, <nav>, <main>, <section>, <article>, and <footer> . This improves accessibility and SEO.

Semantic Elements

  • <header>: Top section, usually with logo/title/navigation.

  • <nav>: Defines navigation menus.

  • <main>: Holds the primary content (only one per page).

  • <section>: Groups related content.

  • <article>: Independent content like blog posts, cards, etc.

  • <footer>: Bottom area with copyright/contact info.

Basic Page Structure

<header>
  <h1>Website Title</h1>
  <nav>...</nav>
</header>

<main>
  <section>
    <h2>Section Title</h2>
    <article>...</article>
  </section>
</main>

<footer>
  <p>Copyright Info</p>
</footer>

Remember: There should be only one <main> element per page.

What is Div, Span, Class & ID in HTML? üßä

HTML provides generic containers <div> and <span> that are styled or targeted with class and id attributes.

Div and Span Elements

  • <div>: Block-level container for grouping content.

  • <span>: Inline container for styling small text portions.

Class and ID Attributes

  • Class: Reusable selector for multiple elements.

  • ID: Unique identifier for one specific element.

Examples

Div Example:

<div>
  <h2>Group Title</h2>
  <p>Some content</p>
</div>

Span Example:

<p>This is <span>important</span> text.</p>

Class vs ID Example:

<div class="product-card">Product 1</div>
<div class="product-card" id="featured-product">Product 2</div>
<div class="product-card">Product 3</div>

Remember: IDs must be unique; classes can be reused.

What is a Multi-page Website? üìí

A multi-page website is a site with multiple HTML files linked together. Each page has consistent navigation to connect them.

Navigation and Linking

  • <nav>: Contains menus for navigating between pages.

  • <a>: The anchor tag, used to create links.

Example Navigation

<nav>
  <ul>
    <li><a href="index.html">Home</a></li>
    <li><a href="about.html">About</a></li>
    <li><a href="contact.html">Contact</a></li>
  </ul>
</nav>
